# GovAI 离线部署说明

## 适用范围

- 适用于与当前服务器相同 CPU 架构的新服务器
- 适用于目标服务器不能联网，需通过 U 盘、移动硬盘等介质离线部署
- 适用于同时迁移代码、Docker 镜像、数据库数据、上传文件

## 离线包包含内容

- 项目代码包：`GovAI_*.tar.gz`
- Docker 运行镜像：`images/*.tar.gz`
- 运行态数据：`runtime_state/govai_db.dump`、`runtime_state/uploads/`
- 环境变量文件：`env/.env.production`、`env/.env.production.example`
- 校验文件：`SHA256SUMS.txt`

## 源服务器打包

在当前服务器项目根目录执行：

```bash
cd /root/govai/GovAI
bash deploy/export-offline-bundle.sh
```

执行完成后，会在 `release/` 目录生成：

```bash
offline_bundle_YYYYMMDD_HHMMSS/
offline_bundle_YYYYMMDD_HHMMSS.tar.gz
```

建议优先传输 `offline_bundle_*.tar.gz` 到离线服务器。

## 目标服务器前置条件

- 已安装 Docker 与 Docker Compose
- 磁盘空间足够容纳镜像包、代码包和业务数据
- 目标服务器端口规划与 `.env.production` 一致

## 目标服务器部署步骤

### 1. 复制离线包并解压

```bash
mkdir -p /opt/govai
tar -xzf offline_bundle_YYYYMMDD_HHMMSS.tar.gz -C /opt/govai
cd /opt/govai/offline_bundle_YYYYMMDD_HHMMSS
```

### 2. 解压项目代码

```bash
tar -xzf artifacts/GovAI_*.tar.gz -C /opt/govai
```

解压后项目目录应为：

```bash
/opt/govai/GovAI
```

### 3. 放置生产环境变量

若离线包中已包含 `env/.env.production`，可直接复制：

```bash
cp env/.env.production /opt/govai/GovAI/.env.production
```

若目标服务器 IP、端口、Dify 地址或密钥不同，请先编辑后再复制。

### 4. 导入 Docker 镜像

```bash
cd /opt/govai/GovAI
bash deploy/load-docker-images.sh /opt/govai/offline_bundle_YYYYMMDD_HHMMSS/images
```

### 5. 启动服务

```bash
cd /opt/govai/GovAI
bash deploy/offline-deploy.sh
```

该脚本会使用：

- `docker-compose.prod.yml`
- `.env.production`
- `docker compose up -d --no-build`

因此不会在离线服务器上重新联网构建镜像。

### 6. 导入业务数据

若需要把当前服务器上的数据库和上传文件一并迁移到新服务器，再执行：

```bash
cd /opt/govai/GovAI
bash deploy/import-runtime-state.sh /opt/govai/offline_bundle_YYYYMMDD_HHMMSS/runtime_state
```

该脚本会：

- 导入 `govai_db.dump`
- 导入 `uploads/`
- 重启 `backend`、`frontend`

## 验证项

部署完成后建议检查：

```bash
docker compose -p govai --env-file .env.production -f docker-compose.prod.yml ps
curl http://127.0.0.1/health
curl -I http://127.0.0.1/docs
```

## Dify / 大模型注意事项

- 当前项目的 AI 能力依赖 Dify 接口
- 若目标服务器没有同步部署 Dify 或没有等价的 API 服务，系统可启动，但 AI 起草、审查、问答等功能无法正常工作
- 若仅做界面或基础链路验收，可临时把 `.env.production` 中的 `DIFY_MOCK` 改为 `true`

## 安全注意事项

- `env/.env.production` 可能包含真实密钥，应仅通过受控介质传输
- 离线包中包含数据库与上传文件，传输和存储时应按生产数据管理
- 若目标服务器与当前服务器网络、端口规划不同，部署前务必修改 `.env.production`
